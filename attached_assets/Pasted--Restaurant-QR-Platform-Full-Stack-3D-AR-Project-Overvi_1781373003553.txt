# Restaurant QR Platform — Full Stack + 3D AR

## Project Overview
Build a **production-grade, ultra-professional restaurant management platform** with three parts:
1. **Customer Mobile App** (PWA — Android & iOS via browser)
2. **Admin Dashboard** (React + Vite — web)
3. **3D AR Viewer** per dish using real AI-generated `.glb` models

All three share a single **Node.js/Express backend** connected to **Neon PostgreSQL**.

---

## Tech Stack (mandatory, no substitutions)

| Layer | Technology |
|---|---|
| Database | Neon PostgreSQL (`@neondatabase/serverless`) |
| Backend | Node.js + Express + Prisma ORM |
| Admin Frontend | React + Vite + TypeScript + TailwindCSS |

## Database Schema (Neon PostgreSQL)

```sql
CREATE TABLE categories (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  icon VARCHAR(50),
  sort_order INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE dishes (
  id SERIAL PRIMARY KEY,
  category_id INT REFERENCES categories(id) ON DELETE CASCADE,
  name VARCHAR(150) NOT NULL,
  description TEXT,
  price DECIMAL(10,2) NOT NULL,
  image_url TEXT,
  model_glb_url TEXT,
  is_available BOOLEAN DEFAULT TRUE,
  sort_order INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE qr_codes (
  id SERIAL PRIMARY KEY,
  label VARCHAR(100),
  target_url TEXT NOT NULL,
  png_base64 TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

Seed with: 4 categories (Pizza, Sandwich, Boisson, Dessert), 3 dishes each. Use realistic Algerian menu names and prices in DZD.

## Backend API (Node.js + Express)

Base URL: `/api`

**Categories**
- `GET /api/categories` — list all with dish count
- `POST /api/categories` — create
- `PUT /api/categories/:id` — update
- `DELETE /api/categories/:id` — delete

**Dishes**
- `GET /api/dishes?category_id=X` — list by category
- `GET /api/dishes/:id` — full detail (includes model_glb_url)
- `POST /api/dishes` — create
- `PUT /api/dishes/:id` — update
- `DELETE /api/dishes/:id` — delete

**QR Codes**
- `GET /api/qr` — get current QR
- `POST /api/qr/generate` — generate new QR PNG pointing to `MENU_URL` env var
- `GET /api/qr/download` — download PNG file

**Static files**
- Serve `/public/models/*.glb` files statically so model-viewer can load them via URL

**Screen 3 — AR / 3D Viewer**

This is the most important screen. Implement it perfectly:

```jsx
import '@google/model-viewer';

export function ARViewer({ dish }) {
  if (!dish.model_glb_url) {
    return (
      <div className="ar-unavailable">
        <p>Modèle 3D non disponible pour ce plat</p>
      </div>
    );
  }

  return (
    <div style={{ width: '100vw', height: '100vh', background: '#000', position: 'relative' }}>
      <model-viewer
        src={dish.model_glb_url}
        ar
        ar-modes="webxr scene-viewer quick-look"
        camera-controls
        auto-rotate
        auto-rotate-delay="0"
        rotation-per-second="20deg"
        shadow-intensity="1.5"
        shadow-softness="1"
        environment-image="neutral"
        exposure="0.85"
        tone-mapping="commerce"
        style={{ width: '100%', height: '100%' }}
      >
        <button
          slot="ar-button"
          style={{
            position: 'absolute',
            bottom: '32px',
            left: '50%',
            transform: 'translateX(-50%)',
            background: 'linear-gradient(135deg, #C9A84C, #E2B95A)',
            color: '#0F0F0F',
            border: 'none',
            borderRadius: '50px',
            padding: '14px 32px',
            fontSize: '16px',
            fontWeight: '600',
            cursor: 'pointer',
          }}
        >
          Placer sur ma table
        </button>
      </model-viewer>

      <button
        onClick={() => navigate(-1)}
        style={{
          position: 'absolute',
          top: '20px',
          left: '20px',
          background: 'rgba(0,0,0,0.6)',
          backdropFilter: 'blur(10px)',
          border: '1px solid rgba(255,255,255,0.15)',
          borderRadius: '50px',
          color: '#fff',
          padding: '8px 18px',

          cursor: 'pointer',
        }}
      >
        Retour
      </button>

      <div style={{
        position: 'absolute',
        top: '20px',
        right: '20px',
        background: 'rgba(0,0,0,0.6)',
        backdropFilter: 'blur(10px)',

```

## 3D AR Integration — Full Technical Detail

### How the pipeline works (implement the storage + serving)

1. 3D models are generated externally using this AI pipeline:
   - **Step 1**: Real photo of dish → **Meshy.ai** "Image to 3D" → base `.glb` model
   - **Step 2**: Import `.glb` into **Luma AI** → improved lighting + textures
   - **Step 3**: Open in **Blender** + **Dream Textures plugin** (Stable Diffusion) → AI retexture with prompt: `"[dish name], studio food photography, sharp details, 8K PBR texture, soft shadows, appetizing"`
   - **Step 4**: Blender export → glTF 2.0 + Draco compression → final `.glb` file under 5MB

2. The `.glb` file is uploaded to `/public/models/` in the project

3. The admin sets the URL in the dish form: `https://yourapp.replit.app/models/pizza-reine.glb`

4. The customer app loads it via `<model-viewer src={dish.model_glb_url}>`

### Install model-viewer correctly
```bash
npm install @google/model-viewer
```
```jsx
import '@google/model-viewer';
// Then use <model-viewer> as a JSX element
// Add to tsconfig: "types": ["@google/model-viewer"]
```

### model-viewer attributes for best food AR result
```html
ar
ar-modes="webxr scene-viewer quick-look"
camera-controls
auto-rotate
shadow-intensity="1.5"
shadow-softness="1"
environment-image="neutral"
exposure="0.85"
tone-mapping="commerce"
scale="1 1 1"
```

`tone-mapping="commerce"` is critical — it makes food colors appear warm, rich, and appetizing instead of flat.

### Fallback handling
```jsx
const [modelError, setModelError] = useState(false);

<model-viewer
  src={dish.model_glb_url}
  onError={() => setModelError(true)}
  ...
/>

{modelError && (
  <div className="model-error">
    Modèle 3D temporairement indisponible
  </div>
)}
```
